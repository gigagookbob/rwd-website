# rwd Landing Page Copy & Structure Rewrite

**Date:** 2026-04-18
**Status:** Design approved, awaiting spec review
**Scope:** Landing page copy rewrite + structural simplification. No visual system changes. No backend changes.

## 1. Why

Current copy on the rwd landing page reads as AI-generated marketing: over-balanced parallels, em-dashes, decorative contrasts like "Metrics show activity. Journals show reasoning.", and six near-identical Showcase blocks. It also fails to position the product against the actual category — **CLI-based AI coding agents** (Claude Code, Codex) rather than web AI.

Two changes at once:

1. **Copy voice.** Rewrite every user-facing string in a direct, developer-blog voice. Strip AI tells.
2. **Structure.** Collapse 6 Showcase sections into 4 Feature cards. Remove the Comparison section. Add a Maker's Note section.

The page should feel like one person wrote it, not a marketing team. The Superset landing page (https://superset.sh/) is the reference for tone and rhythm.

## 2. Goals and non-goals

**Goals**
- Every copy string reads as written by a developer, not generated by an LLM.
- Product positioning is clear on first view: `rwd` is for people using CLI AI agents (Claude Code, Codex, more coming).
- Value prop centers on **decisions / questions / corrections** — the reasoning trail that AI agents strip out.
- Page length drops. Fewer sections, each one earning its place.

**Non-goals**
- No changes to visual system (tokens, typography, spacing, color). DESIGN.md stays as-is.
- No changes to CSS structure beyond what is required to add the Maker's Note section and remove the Comparison section.
- No changes to JS behaviors (FAQ accordion, copy button, fade-in observer).
- No backend/CLI product changes.
- No translation to Korean. Page stays English.

## 3. Copy style rules

These rules apply to every string in the page, including meta tags and JSON-LD.

**Forbidden**
- `—` em-dash. Use period, comma, or line break.
- Parenthetical asides `(like this)` for nuance. Break into a separate sentence or cut.
- Over-balanced triple parallels connected by commas (`what X, what Y, what Z`). If the triple is needed, use fragments: `What X. What Y. What Z.`
- Marketing clichés: `seamlessly`, `effortlessly`, `powerful`, `supercharge`, `transform`, `unlock`, `elevate`.
- Closing hooks appealing to the reader (`If you're someone who...`, `You probably have this problem too`). Make statements, not appeals.
- Over-neat contrast constructions (`X didn't help. They showed A, not B.`). Too crafted.

**Preferred**
- Short fragments. Periods over commas over dashes.
- First-person `I` statements in the Maker's Note only.
- Concrete situation over abstract value.
- Stop before the clincher. Don't wrap.

## 6. Scope boundaries

**In scope**
- `index.html` markup and copy.
- CSS additions for: (a) the new Maker's Note section, (b) the simpler 4-card Feature grid, (c) the Footer CTA sub-line element. Prefer reusing existing `.comparison-card` styles as the base for Feature cards.
- CSS deletions for unused Showcase and Comparison styles in `css/03-components.css` and `css/04-showcase.css` where those styles become unreferenced after the HTML change.
- `sitemap.xml` last-modified date.

**Out of scope**
- `DESIGN.md` update. The design tokens, typography, and color system are unchanged. No update needed.
- `CLAUDE.md`, `AGENTS.md` updates.
- Image assets (terminal screenshot, OG image, favicon). Alt text copy changes but the image files don't.
- JS behaviors.
- CLI product changes or new features.

## 8. Acceptance criteria

- Every copy string on the page matches a string listed in section 5. No leftover Showcase or Comparison copy remains.
- No em-dash (`—`) appears anywhere in `index.html`.
- Grep for the forbidden marketing words in section 3 (`seamlessly`, `effortlessly`, `powerful`, `supercharge`, `transform`, `unlock`, `elevate`) returns no hits in `index.html`.
- Section count on the page: Hero, Terminal Demo, Feature cards, Maker's Note, Install, FAQ, Footer CTA. Seven main sections inside `<main>`.
- FAQ has exactly five items.
- JSON-LD FAQPage has exactly five entries matching the visible FAQ.
- Mobile at 640px renders each section without horizontal scroll.
- Lighthouse accessibility score is no worse than before.
